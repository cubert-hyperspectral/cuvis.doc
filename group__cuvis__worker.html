SDK_CAPI CUVIS_STATUS SDK_CCALL cuvis_worker_create ( CUVIS_WORKER *  o_pWorker,
CUVIS_WORKER_SETTINGS  worker_settings 
)

Create a Worker.

The encapsulates the functions of the Acquisiton Context, Processing Context, Exporter, and Viewer into a single container and manages the communications between these. It also enables multi-threaded operation

Note
The set functions need to be called in order for the worker to be enabled.
Parameters
[out]o_pWorkerThe worker handle to be created
[in]worker_settingsThe worker configuration

◆ cuvis_worker_get_next_result()

SDK_CAPI CUVIS_STATUS SDK_CCALL cuvis_worker_get_next_result ( CUVIS_WORKER  i_worker,
CUVIS_MESU *  o_pMesu,
CUVIS_VIEW *  o_pView,
CUVIS_SIZE  i_Timeout_ms 
)

Get the next result in order.

The measurement will be readyly recorded, processed (if set), stored (if set) and have a view (if set).

Parameters
[in]i_workerThe worker handle
[out]o_pMesuThe recorded measurement or NULL if recording failed
[out]o_pViewThe view, if calculated sucessfully, else NULL
[in]i_Timeout_msThe number of milliseconds to wait for a result. -1 to wait indefinitely
Returns
status_ok or on error: status_error, status_not_processed, status_not_stored, or status_no_view, or status_not_available
